Brojevi od a do b

Motivacija

Zamislite da želite da na tabli ispišete sve brojeve od jednog do drugog, na primer od 3 do 6. Uradili biste to tako što napišete 3, zatim sledeći broj 4, onda 5, i na kraju 6. Jednostavno, zar ne? Ali, kako kompjuter da „nauči“ da uradi isti zadatak, bez obzira koja dva broja zadamo?

Naivno rešenje

"Treba da ispišem sve brojeve od a do b, pa ću napraviti for petlju". Ovo je proceduralno razmišljanje — ne postoji nikakva konstrukcija rešenja, već samo direktno izvršavanje. Na primer, ako je a=3 i b=6, jednostavno napišemo 3,4,5,6. Problem je u tome što ovo rešenje ne razvija algoritamsko razmišljanje, ne postoji razumevanje odnosa između manjih i većih instanci problema i ne uči se kako se rešenje gradi — samo kako se izvršava.

static void Ispisi(int a, int b) { for (int i = a; i <= b; i++) { Console.WriteLine(i); } } // Primer poziva: Ispisi(3, 6);
Naivna rešenja su važna jer nam pokazuju zašto je potrebno bolje razmišljanje.

Analiza problema

Problem možemo posmatrati ovako: da bismo ispisali sve brojeve od a do b, potrebno je da:

  • Ispišemo broj a
  • Zatim ispišemo brojeve od a+1 do b

Dakle, veliki zadatak razlažemo na manji, vrlo sličan zadatak. Ovo je suština induktivnog razmišljanja: kako od rešenja manjeg problema doći do rešenja većeg.

Induktivni pristup

Induktivni pristup, kao što smo rekli, podrazumeva logički proces zaključivanja od pojedinačnog ka opštem, pri čemu se na osnovu pojedinačnih slučajeva donosi opšti sud Osnovni slučaj: ako je a = b, tada ispisujemo samo jedan broj, i time se zadatak završava.
Induktivni korak: ako znamo kako ispisati sve brojeve od a+1 do b, onda je dovoljno da pre toga ispišemo broj a. Tako dobija­mo jasan metod za konstrukciju algoritma.

Primeri i testovi

static void Ispisi(int a, int b) { if (a > b) return; Console.WriteLine(a); Ispisi(a + 1, b); } // Primer poziva: Ispisi(3, 6); // Ulaz 3 6 // Izlaz 3 4 5 6
Ovaj kod pokazuje suštinu induktivno-rekurzivne konstrukcije: prvo rešavamo osnovni slučaj (kada više nema brojeva), a zatim gradimo rešenje većeg problema iz manjeg.

Pitanja za samoevaluaciju

  • Šta znači osnovni slučaj u ovom problemu?
  • Zašto nije dobro rešenje unapred ispisati sve brojeve ručno?
  • Kako bi izgledao algoritam za ispis brojeva od b do a (u opadajućem redosledu)?

Zadaci za vežbu

Napišite algoritam koji ispisuje sve cele brojeve u intervalu [a,b] tako da su brojevi ispisani unazad (od b ka a).
