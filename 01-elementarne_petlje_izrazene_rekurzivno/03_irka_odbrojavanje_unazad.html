Odbrojavanje unazad

Motivacija

Da li ste nekada odbrojavali pred početak trke ili igre? Na primer: „3, 2, 1, 0 – KREĆI!“. Ovaj jednostavan postupak nam pokazuje ideju postepenog smanjivanja sve dok ne dođemo do osnovnog slučaja – nule. Hajde da vidimo kako možemo naučiti računar da uradi isto.

Naivno rešenje

Jedan učenik bi mogao da napiše poseban Console.WriteLine() za svaki broj: 3, 2, 1, 0. To radi ako je broj mali, ali nije praktično ako počinjemo, recimo, od 99 ili nekog drugog većeg broja. Potrebno je pronaći opštije rešenje.

static void Naivno() { Console.WriteLine(3); Console.WriteLine(2); Console.WriteLine(1); Console.WriteLine(0); }
Naivno rešenje nam pokazuje problem ručnog pristupa – nema fleksibilnosti i ne radi za proizvoljan ulaz.

Analiza problema

Zadatak možemo opisati ovako:

  • Ispišemo početni broj a
  • Smanjimo ga za 1 i ponovimo postupak
  • Zaustavimo se kada dođemo do nule

Dakle, da bismo ispisali sve brojeve od a do nule, dovoljno je da najpre ispišemo a, a zatim odbrojimo unazad od a-1.

Induktivni pristup

Osnovni slučaj: ako je broj manji od 0, prekidamo postupak. Induktivni korak: ako je broj a nenegativan, najpre ga ispišemo, a zatim pozovemo isti postupak za a-1.

Primeri i testovi

static void Odbrojavanje(int a) { if (a >= 0) { Console.WriteLine(a); Odbrojavanje(a - 1); } } static void Main() { int a = int.Parse(Console.ReadLine()); Odbrojavanje(a); }
Ovde vidimo jasan induktivni obrazac: ako znamo da odbrojimo od a-1 do nule, onda lako možemo da odbrojimo i od a.

Pitanja za samoevaluaciju

  • Koji je osnovni slučaj u ovom zadatku?
  • Kako se razlikuje odbrojavanje unazad od zadatka „Brojevi od a do b“?
  • Kako bismo odbrojavali unazad, ali samo parne brojeve?

Zadaci za vežbu

Napišite algoritam koji odbrojava unazad od zadatog broja a, ali samo ispisuje brojeve deljive sa 3. Na primer, ako je a = 10, izlaz treba da bude: 9, 6, 3, 0.
