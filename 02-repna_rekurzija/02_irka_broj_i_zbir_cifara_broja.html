Broj i zbir cifara broja

Motivacija

Kada posmatramo broj, često nas interesuju njegove cifre – njihov broj i njihov zbir. Na primer, ako imamo broj 1234, on ima 4 cifre, a njihov zbir je 10. Ovakvi zadaci su idealni za vežbanje induktivnog razmišljanja jer broj možemo posmatrati kao niz manjih delova – njegovih cifara.

Naivno rešenje

Jedan način bio bi da broj pretvorimo u string, zatim prolazimo kroz sve karaktere i sabiramo njihove numeričke vrednosti. Iako funkcionalno tačno, ovde ne koristimo algoritamsko razmišljanje o broju kao matematičkoj celini. Naš cilj je da rešenje izvedemo induktivno, bez oslanjanja na tekstualne operacije.

Analiza problema

Posmatrajmo broj n. Ako brojimo cifre induktivno, broj i zbir cifara broja n jednaki su broju i zbiru cifara broja n/10, uvećanih za doprinos poslednje cifre n % 10. Dakle, možemo reći:

Osnovni slučaj — kada je n = 0, broj cifara i zbir cifara su 0.

Induktivni korak — ako znamo broj i zbir cifara broja n/10, onda samo dodajemo još jednu cifru (n % 10).

Pravilan pristup

Rešenje možemo ostvariti jednostavnom rekurzijom koja računa od najmanjeg ka većem problemu. Koristićemo dve funkcije — jednu koja broji cifre, i drugu koja računa zbir cifara.

int BrojCifara(int n) { if (n == 0) return 0; return 1 + BrojCifara(n / 10); } int ZbirCifara(int n) { if (n == 0) return 0; return (n % 10) + ZbirCifara(n / 10); } // Ulaz: ceo broj n // Izlaz: broj cifara i zbir cifara

Ove dve funkcije ilustruju isti obrazac razmišljanja: polazimo od jednostavnog slučaja (n = 0), zatim u svakom koraku rešavamo manji deo broja, sve dok ne obradimo sve cifre.

Zadaci za vežbu

1. Napiši funkciju koja vraća zbir parnih cifara broja.
2. Napiši funkciju koja računa proizvod cifara broja.
3. Napiši funkciju koja proverava da li broj ima više parnih ili neparnih cifara.
4. Objasni kako se induktivni pristup primenjuje u svakom od zadataka.

Pomoć

Induktivni pristup znači da problem svodimo na jednostavniji oblik. Kada se broj deli sa 10, uklanjamo poslednju cifru, pa svaka sledeća rekurzija obrađuje kraći broj. Kada dostignemo n = 0, vraćamo se kroz sve korake sabirajući ili brojeći cifre.
