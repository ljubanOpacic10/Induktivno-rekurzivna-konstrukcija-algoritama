Jednakost rastojanja

Motivacija

Zamisli da trebaš da pronađeš tačku koja je „jednako udaljena“ od svih datih tačaka na brojevnoj osi. U matematici se takva tačka naziva aritmetička sredina. Na primer, ako imamo tačke sa koordinatama 2, 4 i 6 — tačka koja se nalazi tačno u sredini između svih njih je 4. Ovaj primer nas vodi do ideje da zbir svih tačaka podeljen sa njihovim brojem daje tu „središnju tačku“.

Naivno rešenje

Jedan jednostavan način je da koristimo petlju koja sabira sve unete brojeve i potom podelimo taj zbir sa njihovim brojem. Iako jednostavno, ovaj pristup ne otkriva induktivni obrazac — ne pokazuje kako se zbir postepeno gradi.

Analiza problema

Ako želimo da izračunamo zbir n brojeva, možemo problem posmatrati induktivno:
Zbir prvih n brojeva jednak je zbiru prvih n−1 brojeva uvećan za poslednji broj. Na taj način, ponavljanjem ovog postupka, postepeno dolazimo do ukupnog zbira.

Osnovni slučaj — kada je n = 0, zbir je 0 jer nema brojeva za sabiranje.

Induktivni korak — ako znamo zbir prvih n−1 brojeva, onda se zbir n brojeva dobija dodavanjem još jednog broja.

Pravilan pristup

Koristićemo repno-rekurzivnu funkciju koja postepeno sabira učitane brojeve. Funkcija ima dva parametra:

n — broj preostalih brojeva koji treba da se učitaju,

zbir — trenutni zbir svih dosad učitanih brojeva.

Kada n postane 0, znači da smo učitali sve brojeve i možemo vratiti konačan zbir.

double ZbirUcitanihBrojeva(int n, double zbir) { if (n == 0) return zbir; double x = double.Parse(Console.ReadLine()); return ZbirUcitanihBrojeva(n - 1, zbir + x); } double ZbirUcitanihBrojeva(int n) { return ZbirUcitanihBrojeva(n, 0.0); } // Ulaz: broj n i n realnih brojeva // Izlaz: aritmetička sredina unetih brojeva

Tok izvršavanja možemo pratiti na primeru:
n = 3
Uneti brojevi: 2, 4, 6

ZbirUcitanihBrojeva(3, 0) => učitavamo prvi broj 2, zbir postaje 0 + 2 = 2
ZbirUcitanihBrojeva(2, 2) => učitavamo sledeći broj 4, zbir postaje 2 + 4 = 6
ZbirUcitanihBrojeva(1, 6) => učitavamo poslednji broj 6, zbir postaje 6 + 6 = 12
ZbirUcitanihBrojeva(0, 12) => n = 0, više nema brojeva za unos, rezultat = 12

Dakle, funkcija se završava kada je n = 0 i vraća konačan zbir = 12. Da bismo pronašli aritmetičku sredinu, delimo zbir sa brojem elemenata: 12 / 3 = 4. To znači da je aritmetička sredina unetih brojeva (2, 4, 6) jednaka 4.

Pitanja za samoproveru

Šta se dešava u funkciji kada je n = 0?

Kako funkcija „pamti“ prethodno učitane brojeve?

Zašto koristimo zbir kao parametar koji se prenosi kroz pozive?

Kako bi izgledalo iterativno rešenje istog problema?

Zadaci za vežbu

1. Napiši funkciju koja računa geometrijsku sredinu n brojeva.
2. Napiši funkciju koja računa zbir kvadrata n brojeva.
3. Prilagodi funkciju tako da radi sa celobrojnim nizom umesto da brojevi dolaze sa tastature.
4. Objasni kako se u svim ovim zadacima koristi induktivni pristup.

Pomoć

Kao i u prethodnim zadacima, ključ induktivnog pristupa je u prepoznavanju odnosa između manjih i većih problema. Zbir prvih n brojeva zavisi od zbira prvih n−1 brojeva, a funkcija se završava kada više nema preostalih elemenata. Na taj način dobijamo rezultat postepeno — što je upravo suština induktivno-rekurzivne konstrukcije algoritma.
