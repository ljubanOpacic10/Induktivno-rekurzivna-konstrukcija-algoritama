Jednakost rastojanja

Motivacija

Zamisli da pokušavaš da pronađeš tačku koja je „tačno u sredini“ između svih datih tačaka na brojevnoj osi. Ako staviš te tačke na vagu, tačka ravnoteže bila bi ona u kojoj je ukupna težina sa leve i desne strane ista. U matematici, ta tačka se naziva aritmetička sredina.

Na primer, ako imamo tačke 2, 4 i 6 — tačka koja se nalazi tačno između njih je 4. Do nje dolazimo tako što saberemo sve vrednosti i podelimo sa njihovim brojem: (2 + 4 + 6) / 3 = 4. Ova ideja je važna, jer se koristi u svemu — od fizike i ekonomije do mašinskog učenja.

Naivno rešenje

Najjednostavnije je koristiti petlju: sabiramo sve brojeve i na kraju podelimo sa njihovim brojem. Ovaj pristup je ispravan, ali ne otkriva način razmišljanja koji stoji iza rešenja. Da bismo razvili algoritamsko mišljenje, pokušaćemo da pronađemo induktivni obrazac.

Analiza problema

Zamislimo da imamo n brojeva. Ako znamo zbir prvih n−1 brojeva, do ukupnog zbira svih n dolazimo jednostavno: dodamo još poslednji broj. Dakle:

Zbir(n) = Zbir(n−1) + poslednji_broj

Na taj način, rešenje većeg problema (sabiranje n brojeva) gradimo iz rešenja manjeg problema (sabiranje n−1 broja). To je upravo induktivni pristup.

Osnovni slučaj — kada je n = 0, zbir je 0 jer nema brojeva za sabiranje.

Induktivni korak — ako znamo zbir prvih n−1 brojeva, zbir n brojeva dobijamo dodavanjem novog broja.

Pravilan pristup

Koristićemo repno–rekurzivnu funkciju koja postepeno sabira brojeve dok ih korisnik unosi. Funkcija ima dva parametra:

n — broj preostalih brojeva koji treba da se učitaju,

zbir — trenutni zbir svih do sada učitanih brojeva.

Kada n postane 0, znači da smo uneli sve brojeve i možemo vratiti konačan zbir.

double ZbirUcitanihBrojeva(int n, double zbir) { if (n == 0) return zbir; // osnovni slučaj double x = double.Parse(Console.ReadLine()); return ZbirUcitanihBrojeva(n - 1, zbir + x); // induktivni korak } double ZbirUcitanihBrojeva(int n) { return ZbirUcitanihBrojeva(n, 0.0); } // Ulaz: broj n i n realnih brojeva // Izlaz: aritmetička sredina unetih brojeva

Tok izvršavanja

Za n = 3 i brojeve 2, 4, 6:

  • ZbirUcitanihBrojeva(3, 0) → unos: 2 → novi zbir = 2
  • ZbirUcitanihBrojeva(2, 2) → unos: 4 → novi zbir = 6
  • ZbirUcitanihBrojeva(1, 6) → unos: 6 → novi zbir = 12
  • ZbirUcitanihBrojeva(0, 12) → kraj rekurzije → vraća 12

Dakle, zbir je 12, a aritmetička sredina se dobija kao 12 / 3 = 4. To znači da je tačka jednako udaljena od svih unetih brojeva.

Pitanja za samoproveru

  • Šta funkcija vraća kada je n = 0?
  • Kako funkcija „pamti“ prethodni zbir?
  • Zašto zbir prolazi kroz funkciju kao parametar?
  • Da li bi rezultat bio isti ako bismo koristili petlju umesto rekurzije?

Zadaci za vežbu

1. Napiši funkciju koja računa geometrijsku sredinu n brojeva.

2. Napiši funkciju koja računa zbir kvadrata n brojeva.

3. Prilagodi funkciju tako da radi sa nizom brojeva umesto unosom sa tastature.

4. Objasni kako se u svim ovim slučajevima primenjuje induktivni korak.

Pomoć – Jednakost rastojanja

Kao i u prethodnim primerima, rešenje se gradi korak po korak. Zbir prvih n brojeva zavisi od zbira prvih n−1, i tako sve dok ne dođemo do osnovnog slučaja (n = 0). Kada se rekurzija „vrati unazad“, rezultat predstavlja ukupni zbir svih brojeva.

Na kraju samo podelimo zbir sa brojem unetih elemenata i dobijamo aritmetičku sredinu — tačku koja je jednako udaljena od svih vrednosti.
